## 제품 가치

### V1: 세션 격리
- **유형**: 구체적
- **설명**: 모든 세션은 전용 data plane pod에서 독립적으로 실행되어, 한 세션의 장애·부하·보안 이슈가 다른 세션으로 전파되지 않는다. (1 세션 = 1 pod)

### V2: 유휴 자원 회수
- **유형**: 구체적
- **설명**: 최대 60분 이상 사용되지 않은 세션은 CRIU 스냅샷으로 동결하고 pod 자원을 회수하여, 활성 세션에만 컴퓨팅 자원을 집중한다.

### V3: 끊김 없는 세션 연속성
- **유형**: 추상적
- **설명**: 스냅샷으로 동결된 세션도 다시 접근하면 CRIU 복원을 통해 동결 직전 상태 그대로 재개된다. 사용자는 세션이 멈췄다는 사실을 거의 인지하지 못한 채 작업을 이어간다.

### V4: 자유로운 멀티세션 전환
- **유형**: 추상적
- **설명**: 사용자는 보유한 여러 세션 사이를 제약 없이 오갈 수 있으며, 전환 대상 세션의 상태(active/idle/snapshot)와 무관하게 일관된 경험으로 접근한다.

### V5: 일관된 세션 상태
- **유형**: 구체적
- **설명**: 세션 상태 전이와 동시 접근은 ConfigMap(resourceVersion CAS) + Lease 기반 atomic operation으로 처리되어, 경쟁 상황에서도 세션 상태가 깨지거나 중복 처리되지 않는다.

### V6: 인터랙티브 쉘 세션 (핵심 산출물)
- **유형**: 구체적
- **설명**: 하나의 세션은 전용 data plane pod 안에서 지속 실행되는 **인터랙티브 쉘**(기본 `/bin/bash`, PTY에 연결)이다. 사용자는 write API로 쉘의 stdin(입력: 명령/키 입력)에 쓰고, read API로 쉘의 stdout/stderr(출력)를 읽는다. 쉘의 프로세스 트리 — 환경 변수, 현재 작업 디렉터리, 쉘 변수·함수·alias, 실행 중인 포그라운드 자식 프로세스, 열린 파일 디스크립터 — 가 곧 세션의 "상태"이며, 이 상태가 idle·snapshot·restore를 거쳐도 동결 직전 그대로 이어진다.
- **비고**: V6는 **"세션이 무엇인가"**를 정의하는 기반 가치이고, V1~V5는 이 쉘 세션을 격리(V1)·회수(V2)·복원(V3)·전환(V4)·일관 처리(V5)하기 위한 품질 속성이다. 즉 V1~V5는 V6를 안전하고 끊김 없이 제공하기 위해 존재한다.
