## 제품 가치

### V1: 세션 격리
- **유형**: 구체적
- **설명**: 모든 세션은 전용 data plane pod에서 독립적으로 실행되어, 한 세션의 장애·부하·보안 이슈가 다른 세션으로 전파되지 않는다. (1 세션 = 1 pod)

### V2: 유휴 자원 회수
- **유형**: 구체적
- **설명**: 최대 60분 이상 사용되지 않은 세션은 CRIU 스냅샷으로 동결하고 pod 자원을 회수하여, 활성 세션에만 컴퓨팅 자원을 집중한다.

### V3: 끊김 없는 세션 연속성
- **유형**: 추상적
- **설명**: 스냅샷으로 동결된 세션도 다시 접근하면 CRIU 복원을 통해 동결 직전 상태 그대로 재개된다. 사용자는 세션이 멈췄다는 사실을 거의 인지하지 못한 채 작업을 이어간다.

### V4: 자유로운 멀티세션 전환
- **유형**: 추상적
- **설명**: 사용자는 보유한 여러 세션 사이를 제약 없이 오갈 수 있으며, 전환 대상 세션의 상태(active/idle/snapshot)와 무관하게 일관된 경험으로 접근한다.

### V5: 일관된 세션 상태
- **유형**: 구체적
- **설명**: 세션 상태 전이와 동시 접근은 ConfigMap(resourceVersion CAS) + Lease 기반 atomic operation으로 처리되어, 경쟁 상황에서도 세션 상태가 깨지거나 중복 처리되지 않는다.
