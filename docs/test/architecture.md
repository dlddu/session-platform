# 테스트 문서: 세션 아키텍처 & 격리

## 검증 대상 AC
- AC-A1: Control plane / data plane 분리 (PRD: 세션 아키텍처 & 격리)
- AC-A2: 세션당 전용 Pod (PRD: 세션 아키텍처 & 격리)
- AC-A3: 세션 종료·동결 시 자원 회수 (PRD: 세션 아키텍처 & 격리)

## 테스트 시나리오

### 시나리오 1: 세션 생성 시 data plane pod 기동
- **사전 조건**: control plane 정상 기동, 클러스터에 가용 자원 존재
- **실행 단계**: control plane API로 세션 1개 생성 요청
- **기대 결과**: 전용 data plane pod 1개가 기동되고, control plane 프로세스 내에는 세션 워크로드가 없음
- **검증 AC**: AC-A1

### 시나리오 2: 세션-Pod 1:1 매핑 및 격리
- **사전 조건**: 클러스터 정상
- **실행 단계**: 세션 3개 생성 → pod 목록 확인 → 세션 2의 pod를 강제 종료
- **기대 결과**: 고유 pod 3개 존재(1:1), 세션 2 종료가 세션 1·3에 영향을 주지 않음
- **검증 AC**: AC-A2

### 시나리오 3: 동결 시 자원 회수
- **사전 조건**: active 세션 1개와 그 pod가 존재
- **실행 단계**: 해당 세션을 스냅샷으로 동결(또는 종료)
- **기대 결과**: 대상 pod가 제거되고 클러스터 가용 자원이 회복됨
- **검증 AC**: AC-A3
