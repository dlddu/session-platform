> 백엔드 측 사슬(`가치 → PRD → AC → 테스트`)은 별도 문서 `doc-tracker.md`에서 추적합니다.

## 현재 상태 요약

- 정의된 가치: **5개** (V1~V5, `values.md` — 참조 전용)
- 사용자 여정: **4개** (J1~J4) — 전부 가치에 연결됨 (고아 여정 0)
- 여정 단계: **14개** (J1: 3, J2: 4, J3: 4, J4: 3)
- Mockup: **4개** (`mockups/`: index, new-session, workspace, restore) — 매핑 단일 소스는 `mockups/README.md`(인덱스)
- 디자인 시스템: **미정의** (토큰/컴포넌트/패턴 없음)
- 주입형 스킬(`.claude/skills/`): **미설치**
- **건강 상태**: 🟡 **부분 시각화 + 디자인 시스템 부재** — J1은 완전 시각화, J2·J3는 부분, J4는 백엔드 여정이라 의도적 비시각화. mockup 4종이 모두 디자인 시스템 없이 임의 스타일.

## 위험 진단

### 🔴 가치 측 위험
- **고아 여정**: 없음 (J1~J4 모두 유효한 가치 참조)

- **가치 미정의**: 해당 없음

### 🟡 시각화 누락
- **시각화 누락 단계**: `J2-S3`(재접근), `J3-S4`(A로 복귀) — 전용 화면 없음. 의도적 제외 여부 결정 필요.
- **부분 시각화 단계**: `J2-S1`, `J2-S2`, `J3-S2`, `J3-S3` — 상태 표시·네비게이션·전환 동작으로 암시되나 단계 전용 화면 없음.
- **시각화 없는 가치**: 없음 (V1~V5 전부 연결).

### 🟢 디자인 시스템 측 위험
- **임의 스타일 mockup**: index/new-session/workspace/restore **4개 전부**. 디자인 시스템 부재로 인라인 토큰 사용. 디자인 시스템 셋업(진입점 5) 후 마이그레이션 또는 시스템 편입 필요.

### 🟠 인접 사슬에서 인지된 항목 (이 스킬 관할 밖, 참고용)

## 수용된 위험
- **J4 시각화 제외 (2026-06-27 수용)**: J4는 백엔드 동시성 여정(ConfigMap resourceVersion CAS + Lease 기반 atomic 전이)으로 **UI 시각화 대상이 아님**. 따라서 J4-S1~S3의 "mockup 없음"은 누락이 아니라 의도된 상태. 이전에 `mockups/restore.html`·`workspace.html`에 있던 동시 접근(operator+automation) 패널은 UI 비대상 판단으로 제거. 사유: 그릴 화면이 없는 백엔드 일관성 보장.
