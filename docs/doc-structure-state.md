# Session Pod Platform 디자인 문서 구조 상태 추적

> 이 문서는 **프론트엔드/디자인 측 사슬**(`가치 → 사용자 여정 → mockup ↔ 디자인 시스템`)의 상태를 추적합니다.
> 백엔드 측 사슬(`가치 → PRD → AC → 테스트`)은 별도 문서 `doc-tracker.md`에서 추적합니다.
> 두 문서가 공유하는 단일 진실 원천은 가치 문서(`values.md`, V1~V6)입니다.

- **마지막 검증 시점**: 2026-07-01
- **검증 범위**: V6(인터랙티브 쉘) 확정에 따른 J5 신설 및 매핑(가치↔여정·여정단계↔mockup·가치↔mockup 매트릭스 갱신). 이어 `workspace.html` session shell 콘솔을 key/value→쉘 명령 기준으로 갱신하여 J5-S1~S3 ✅ 확정. 잔여: J5-S4 미시각화, `restore.html` 콘솔 미갱신(아래 위험)

## 현재 상태 요약

- 정의된 가치: **6개** (V1~V6, `values.md` — 참조 전용)
- 사용자 여정: **5개** (J1~J5) — 전부 가치에 연결됨 (고아 여정 0)
- 여정 단계: **18개** (J1: 3, J2: 4, J3: 4, J4: 3, J5: 4)
- Mockup: **4개** (`mockups/`: index, new-session, workspace, restore) — 매핑 단일 소스는 `mockups/README.md`(인덱스)
- 디자인 시스템: **미정의** (토큰/컴포넌트/패턴 없음)
- 주입형 스킬(`.claude/skills/`): **미설치**
- **건강 상태**: 🟡 **부분 시각화 + 디자인 시스템 부재** — J1은 완전 시각화, J2·J3·J5는 부분(J5는 `workspace.html` 콘솔이 쉘 명령으로 갱신되어 S1~S3 ✅, S4만 미시각화), J4는 백엔드 여정이라 의도적 비시각화. mockup 4종이 모두 디자인 시스템 없이 임의 스타일.

## 위험 진단

### 🔴 가치 측 위험
- **고아 여정**: 없음 (J1~J5 모두 유효한 가치 참조)
- **고아 mockup**: 없음 — mockup 인덱스(`mockups/README.md`)로 4개 전부 여정/가치에 매핑됨
- **가치 미정의**: 해당 없음

### 🟡 시각화 누락
- **시각화 누락 단계**: `J2-S3`(재접근), `J3-S4`(A로 복귀), `J5-S4`(쉘 상태 축적) — 전용 화면 없음. 의도적 제외 여부 결정 필요.
- **부분 시각화 단계**: `J2-S1`, `J2-S2`, `J3-S2`, `J3-S3` — 상태 표시·네비게이션·전환 동작으로 암시되나 단계 전용 화면 없음.
- **mockup 내용 갱신(J5-S1~S3)**: ✅ **해결됨 (2026-07-01)** — `workspace.html`의 session shell 콘솔을 옛 key/value write 예시에서 쉘 명령 입력·stdout 출력(복원 시 env·cwd 보존)으로 갱신. J5-S1~S3 ⚠️→✅.
- **시각화 없는 가치**: 없음 (V1~V6 전부 연결). V6는 `workspace.html`(쉘 콘솔)로 시각화, S4만 미시각화.

### 🟢 디자인 시스템 측 위험
- **임의 스타일 mockup**: index/new-session/workspace/restore **4개 전부**. 디자인 시스템 부재로 인라인 토큰 사용. 디자인 시스템 셋업(진입점 5) 후 마이그레이션 또는 시스템 편입 필요.
- **🟡 restore.html 콘솔 미갱신 (2026-07-01 신규 인지)**: `restore.html`은 복원 후 세션 콘솔을 `workspace.html`과 동일한 옛 **key/value read/write 예시**로 렌더한다(`read {key}`·`write {key,value}`). `restore.html`은 J2-S4(복원)를 시각화하며 V6 직접 대상은 아니나, 세션 정체가 쉘로 확정된 뒤에는 이 콘솔도 쉘 명령/출력으로 갱신해야 mockup suite 전체가 정합. 이번 변경(workspace.html)에는 미포함 — 별도 갱신 필요(web-artifacts-builder 영역).

### 🟠 인접 사슬에서 인지된 항목 (이 스킬 관할 밖, 참고용)
- **고아 가치(소유자 미지정)**: V1~V6 전부 소유자 미지정 — `doc-tracker.md`/`values.md` 참고. product-doc-engineer 영역이라 여기서 수정하지 않음.

## 수용된 위험
- **J4 시각화 제외 (2026-06-27 수용)**: J4는 백엔드 동시성 여정(ConfigMap resourceVersion CAS + Lease 기반 atomic 전이)으로 **UI 시각화 대상이 아님**. 따라서 J4-S1~S3의 "mockup 없음"은 누락이 아니라 의도된 상태. 이전에 `mockups/restore.html`·`workspace.html`에 있던 동시 접근(operator+automation) 패널은 UI 비대상 판단으로 제거. 사유: 그릴 화면이 없는 백엔드 일관성 보장.
