# J4: 동시 접근에서도 깨지지 않는 세션 상태

> 사용자 여정 문서의 일부입니다. 페르소나 정의·가치 커버리지·미해결 항목은 [README.md](./README.md) 참고.

- **페르소나**: P1 + P2 (작업자와 자동화 클라이언트가 같은 세션을 동시에 다룸)
- **상황**: 같은 세션에 거의 동시에 여러 요청이 도달한다 (예: 작업자의 write와 자동화의 read가 겹치거나, 복원과 스냅샷이 동시에 발생).
- **달성 가치**: **V5 일관된 세션 상태** (보조적으로 V3)

> ⚪ **시각화 정책 (의도적 비시각화)**: J4는 동시 접근을 ConfigMap(resourceVersion CAS) + Lease 기반 atomic 전이로 푸는 **백엔드 동시성 여정**으로, UI에 그릴 화면이 없습니다.
> 아래 모든 단계는 *누락이 아니라 의도적으로 mockup 없음*입니다. 페르소나 P1+P2의 동시 접근 시나리오는 단계 시각화의 *행위자*였을 뿐
> 그 자체가 시각화 대상이 아닙니다(시각화 대상은 가치·단계). 이전에 `../mockups/restore.html`·`workspace.html`에 있던
> "Attached clients(operator+automation) + 동시 atomic 전이" 패널은 UI 비대상으로 2026-06-27 제거됨. (매핑: [`../mockups/README.md`](../mockups/README.md))

## 단계

> 단계 표기: `J4-S{n}` · 각 단계 끝의 *mockup* 항목은 시각화 산출물 연결 상태입니다.

1. **J4-S1 · 동시 요청 발생** — 동일 세션에 복원/스냅샷/read/write 요청이 동시 다발로 들어온다. *(mockup: 없음 — 백엔드 동시성, UI 비대상)*
2. **J4-S2 · atomic 전이로 단일화** — ConfigMap resourceVersion compare-and-swap(점유는 Lease)으로 단일 상태 전이만 성공하고, 중복 pod 기동·이중 스냅샷이 발생하지 않는다. *(관련 AC: AC-C1)* *(mockup: 없음 — 백엔드 동시성, UI 비대상)*
3. **J4-S3 · 일관된 결과** — 모든 클라이언트가 유효한 단일 상태로 수렴한 세션을 보며, 상태가 깨지거나 중복 처리되지 않는다. *(관련 AC: AC-C1, AC-C3)* *(mockup: 없음 — 백엔드 동시성, UI 비대상)*
